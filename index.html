Overview

Pico Ultimate is the "ultimate" compact autonomous helper: a minimalistic, high-performance robot that can carry up to 25 kg, pick and place a 20 L water jug, navigate indoor/outdoor environments, climb/descend up to 30° slopes and stairs, and run all day using swappable battery modules. For compute it uses an NVIDIA Jetson Orin 64GB module (Orin-class 64GB variant) to enable on-device large model inference, real-time SLAM, and manipulation planning.

Capabilities (Ultimate Spec)

  • Payload: 25 kg peak (design target), 20 L jug handling standard.
  • Compute: NVIDIA Jetson Orin 64GB module — supports large-model inference (LLMs), multi-camera fusion, and MoveIt2 at high frame rates.
  • Runtime: 12–16 hours typical with 12 kWh usable battery; hot-swap modules for 24/7 operation.
  • Mobility: 4× independently-driven wheels with leg-assist actuators for obstacle negotiation and stair transitions.
  • Perception: 360° LiDAR + stereo depth cameras + RTK-GNSS for outdoor precision.
  • Safety: Redundant E-stop, anti-tip controller, strain gauges and load-cell monitoring of payload.
  • Interfaces: Voice, smartphone app, optional cloud integration, and local web UI for diagnostics.

Mechanical Design & Arm Mechanism

The manipulator is a 7-DOF serial arm with harmonic drives on high-torque joints. The chassis is compact with a recessed tray for the water jug and a kinematic guide for precise placement. Key dimensions and tolerances are included for CNC and 3D printing.

Kinematic keypoints

  • Base yaw rotation: ±180°
  • Shoulder pitch: -90° to +90°
  • Elbow range: -135° to +135°
  • Wrist: full spherical motion with roll ±360°

Joint torque calculation (worked digits)


Payload m = 20.0 kg Gravity g = 9.81 m/s^2 Reach r = 0.5 m F = m * g = 20.0 * 9.81 = 196.2 N Torque_required = F * r = 196.2 * 0.5 = 98.1 N·m Design margin 1.3 => target = 98.1 * 1.3 = 127.53 N·m Round -> 130 N·m target at shoulder joint

Tray & cradle

Design a kinematic cradle with two tapered locating pins and a rear stop. Use a motorized latch and load-cell under the tray to confirm seating. Integrate an RFID reader or dry-contact to signal the dispenser to flow.

Electrical & Power

Battery sizing (exact math)


Assumption: average draw = 800 W (0.8 kW) Runtime target = 12 hours Energy_needed = 0.8 kW * 12 h = 9.6 kWh Buffer (30%) -> 9.6 * 1.3 = 12.48 kWh -> choose 12 kWh usable At 48 V: Ah = Wh / V = 12,000 Wh / 48 V = 250 Ah Pack: 48 V, 250 Ah ~= 12 kWh

Wiring & safety

Main bus = 48 V, with precharge resistor, main contactor and BMS. Use 500 A-rated contactor for high peaks. E-stop cuts contactor coil via redundant circuit. Use automotive-grade M12 connectors for sensors and sealed power connectors.

Power distribution diagram (ASCII)


[Battery 48V] -> [Main Contactor + Precharge] -> [DC Bus Caps] -> [Motor Controllers (CAN)] -> Wheel motors -> [48->12V DC-DC] -> Jetson Orin 64GB + Perception -> [5V rail] -> Sensors & MCUs BMS <-> Jetson via CAN for telemetry

Software Architecture & Examples

Software is ROS2-based (Humble or later). Heavy perception and LLM workloads run on the Jetson Orin 64GB. Use MoveIt2 for manipulation, Nav2 for navigation, and a YOLOv8-family detector for object detection. LIO-SAM or Cartographer for LiDAR SLAM. Use containerized deployment (Docker) for reproducibility.

Model recommendations

  • Object detection: YOLOv8 medium, quantized for Jetson for speed.
  • Segmentation for grasping: small UNet on depth projections.
  • LLM & dialog: run a local LLM (e.g., Mistral, Llama 2 distilled) with GPU offload; use it for high-level instructions, task planning, and natural dialog.

Step-by-step Build Instructions

  1. Frame & chassis: cut and assemble aluminum extrusions; machine base plate for battery trays. Torque M8 structural bolts to 20 N·m.
  2. Wheel modules: mount hub motors, test free-spin, install encoders and suspension links.
  3. Battery & BMS: mount battery modules, connect BMS, test voltage and cell balancing with multimeter on bench.
  4. Manipulator: assemble harmonic drives to links, install motors, route cables through drag chain, mount the wrist F/T sensor and gripper.
  5. Sensors: fix LiDAR on mast with anti-vibration mounts, mount cameras with known extrinsics, secure IMU near CoG.
  6. Software: flash Jetson with JetPack, install ROS2, dockerize stacks, run unit tests for perception and motor control.
  7. Calibration: calibrate camera intrinsics, LiDAR extrinsics, IMU & odometry, run SLAM initialization and store baseline map.
  8. Testing: run unloaded motion tests, verify traction on incline ramp, then test with 20L dummy load. Record currents and temperatures.

Assembly tolerances & fasteners

  • Mating pin tolerance: ±0.05 mm for kinematic guides.
  • Surface finish tolerance for plates: ±0.2 mm.
  • Use thread-locking compound on vibration-critical fasteners; re-torque after 50 hours of operation.

Testing & Validation

Run progressive tests: bench tests, small motion, full navigation in controlled environment, loaded incline tests, and full mission run. Log everything and analyze failures.

  1. Static CoG and tipping test.
  2. Full SLAM mapping and localization consistency checks.
  3. Manipulation repeatability (≥500 place cycles without drift).

Maintenance & Operations

  • Daily: clean sensors, check battery state, run health diagnostics.
  • Weekly: check fasteners, grease gearboxes, inspect belts/straps.
  • Monthly: update models, firmware, check battery capacity.
